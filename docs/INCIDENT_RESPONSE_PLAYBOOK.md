# Incident Response Playbook

1. **Detect and triage:** confirm the signal, severity, affected service/site, and customer or compute impact.
2. **Assign:** name an incident commander and technical owner; record the responsible team and vendor.
3. **Contain:** reduce load, isolate a failed asset, fail over, or pause a pipeline when doing so lowers risk.
4. **Diagnose:** correlate telemetry, changes, audit events, asset history, and vendor status.
5. **Recover:** execute the runbook, monitor recovery indicators, and validate data freshness.
6. **Resolve:** record root cause, action taken, resolved time, impact, and follow-up work.
7. **Review:** calculate MTTR, identify control gaps, and update monitoring or documentation.

Critical incidents target acknowledgment within 10 minutes and vendor escalation within the contracted SLA. Never resolve solely because a metric briefly returned to normal.
